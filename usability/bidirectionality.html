UI 镜像概述

从左到右（LTR）语言和从右到左（RTL）语言的最大区别是内容的显示方向：

  • LTR 语言从左到右显示内容
  • RTL 语言从右到左显示内容

RTL 内容还会影响一些图标和图片的显示方向，尤其是表示事件顺序的图标和图片。

一般来说，对于 LTR 语言，时间流逝方向也是从左到右；对于 RTL 语言，时间流逝方向也是从右向左。

当 UI 从 LTR 更改为 RTL 时（反之亦然），通常称为镜像。RTL 布局是 LTR 布局的镜像。镜像会同时影响布局和图像。

当 UI 从一个方向更改为另一个方向时，这些项目不会镜像：

  • 数字
  • 未翻译的文本（即使它是词组的一部分）

文本方向总是和语言方向保持一致。例如，任何 LTR 语句（例如一个 URL）将继续以 LTR 的格式显示，即使 UI 的其余部分是 RTL 的。

当镜像一个 UI 时，这些元素会发生改变：

  • 文本框图标显示在字段的另一侧
  • 导航按钮指向相反的方向
  • 表示方向的图标会被镜像，例如箭头
  • 文本（如果翻译成 RTL 语言）右对齐

这些元素不会被镜像：

  • 不表示方向的图标，例如相机
  • 数字，例如时钟和电话号码
  • 图表和图解

RTL 镜像指南

在适配从右向左的 UI 时，文本、布局和图标，以及任何和时间相关的元素都应当从右向左移动。例如，前进即为向左移动，后退即为向右移动。

何时需要镜像

对于镜像最重要的图标是前进和后退按钮。后退和前进导航按钮会反转。

表示向前移动的图标应该被镜像。

大多数的 RTL 国家或地区不会镜像反斜杠。在 RTL 中把带反斜杠的图片以原样保留。

在音量图标右侧有一个滑块时，应该被镜像。滑块的进度方向也要改为从右到左，且声波应当朝向左边。

在有些情况下，水平和圆形方向的时间会同时隐含在图标中。例如，Google Docs 中的撤销和重做按钮同时具有水平方向和圆形方向。

在 LTR 中，它们在水平和圆形方向指向相同的时间方向。在 RTL 中，可以只显示水平方向，或只显示圆形方向。

图标内含有指代文本的图形时，在镜像时需要格外仔细。

文字在 RTL 中是右对齐的。如果段落开头有缩进，或段落末尾有未完结的行，再或者段落右侧参差不齐，则图标需要被镜像。

何时不需要镜像

虽然时间的线性流逝方向在 RTL 中需要被镜像，但表示时间方向的圆形不是。在 RTL 语言中，时钟仍然顺时针旋转。时钟图标、或具有顺时针箭头的旋转刷新或进度指示器不应被镜像。

有些图标指代的是现实中的物体，这些图标不需要镜像。

例如，全世界的键盘看起来都是一样的，所以它们不应该被镜像。

某些图标可能看起来具有方向性，但实际上它们表示的是右手持握的物体。

例如，搜索图标通常在右下侧会有手柄，因为大部分用户是右撇子，会用右手来握住放大镜。

在用右手书写的国家中，大部分用户也是右撇子，所以这些图标不应该被镜像。

时间的流逝

任何描述时间流逝的东西应该被镜像。

不要镜像媒体播放按钮和媒体进度指示器，因为它们指的是播放磁带的方向，而不是时间的方向。

本地化

图形中的文本

包含文本的图形通常需要本地化。

数字

包含数字的图标必须本地化为当地语言所使用的数字。例如，孟加拉文、马拉地文、尼泊尔文、和一些阿拉伯文的使用地区，使用的是不同形式的数字。

镜像

即使 UI 没有被镜像，但某些内容也可能需要被镜像。例如，当用户在一个 LTR 的文档中编辑一个 RTL 的段落时，那个特定段落的工具栏按钮应该是 RTL 的。
